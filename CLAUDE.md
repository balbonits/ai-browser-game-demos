# CLAUDE.md

Guidance for Claude Code working in this repo. See also [`AGENTS.md`](AGENTS.md) for tool-agnostic agent guidance and [`docs/`](docs/) for deeper details.

## What this repo is

A showcase of browser games and game demos built entirely by AI. Game code is generated by Claude Code; pixel art is generated by PixelLab.ai (via MCP). The human's role is direction and review.

The repo has **two layers** with different rules:

1. **The landing shell** — root `index.html` + `src/` + Tailwind/Vite, a React + TypeScript app that renders the home page and game card grid. Mirrors the `jdilig-me-v3` stack.
2. **The games** — each lives self-contained under `public/games/<slug>/` as pure static HTML/CSS/JS. **No build step inside a game.** Vite copies `public/` verbatim; the games never know React exists.

Don't conflate the two — the rules below differ for each.

## Ground rules — games (`public/games/<slug>/`)

- **No hand-written game code.** All game code is produced by Claude (or another agent). If the user hand-writes something, they'll note it.
- **Static only, no build step.** A game must run by opening its `index.html` over an HTTP server — no bundler, no transpilation, no `npm install` required to play it.
- **Vanilla by default, engines when they fit.** Most games are plain canvas + JS. **Phaser**, **PixiJS**, and **Three.js** are welcome when a game's scope earns them — load them via CDN (e.g. `esm.sh`, `cdn.jsdelivr.net`). Confirm the engine choice with the user the first time it's used on a given game.
- **Off-limits inside a game folder:** UI frameworks (React/Vue/Svelte), TypeScript, Tailwind, anything that requires a build step. Games stay portable static folders.
- **Self-contained.** Each game owns its assets under top-level category folders (`characters/`, `obstacles/`, `backdrops/`, etc.). Do not reach outside the folder for game-specific code or art.
- **Register new games** by adding an entry to `src/data/games.ts` so the landing page picks it up.

## Ground rules — landing shell (root `index.html` + `src/`)

- **React 19 + TypeScript + Vite + Tailwind 4 + React Router 7.** Mirrors the parent site (`jdilig-me-v3`) so design tokens and components stay aligned.
- **Build via `npm run build`** (`tsc -b && vite build`). Vercel auto-detects Vite. Output goes to `dist/`.
- **Static files** (LICENSE, LICENSE-ASSETS, logo.png, credits.html, the games folders) live in `public/` and are copied verbatim by Vite. The repo-root `LICENSE` and `LICENSE-ASSETS` are surfaced to the dev server + build via the `rootLicenseFiles` plugin in `vite.config.ts` (single source of truth at the repo root).
- **Shell components live in `src/components/site/`**; routes in `src/routes/`; theme tokens in `src/styles/tokens.css`. Don't add a per-page CSS file — use Tailwind utilities + tokens.
- **No new pages without asking.** Right now there's only `/` (Home) plus a static `/credits.html`. Adding routes (about, blog, etc.) is a conversation.

## Adding a new game

See [`docs/adding-a-game.md`](docs/adding-a-game.md) for the step-by-step checklist. TL;DR:

1. Create `public/games/<slug>/index.html` (plus CSS/JS/assets as siblings).
2. Add an entry to `GAMES` in `src/data/games.ts`.
3. Put any PixelLab-generated art under `public/games/<slug>/<category>/`.

## Art pipeline (PixelLab MCP)

The PixelLab MCP is configured for this project. Available tool families:

- `create_character`, `animate_character`, `list_characters`, `get_character`, `delete_character`
- `create_topdown_tileset`, `create_sidescroller_tileset`, `create_tiles_pro`
- `create_isometric_tile`, `create_map_object`
- Plus `get_*`, `list_*`, `delete_*` variants

See [`docs/pixellab.md`](docs/pixellab.md) for workflow details (jobs are async — you get a job ID back and poll, download the resulting image, and save it under the game's `<category>/` folder).

## When in doubt

- Inside a `public/games/<slug>/` folder: proceed freely (vanilla JS, CSS, HTML).
- Adding a CDN engine (Phaser/PixiJS/Three.js) for the first time on a given game: ask once.
- Touching the shell (root `index.html`, `src/`, `tailwind`, `vite.config.ts`): ask before structural changes.
- Adding a new top-level dependency to `package.json`: ask first.
- Adding a build step inside a game folder: don't — propose a different solution first.

## Subagents

Two project-level subagents are defined under `.claude/agents/`:

- **`artist`** — PixelLab MCP asset generation and organization. Delegate anything involving sprites, animations, or tilesets.
- **`dev`** — game code (vanilla / Phaser / PixiJS / Three.js inside `public/games/<slug>/`) and shell code (React / TS / Tailwind inside `src/`). Delegate anything involving HTML/CSS/JS/TSX, game loops, collision, input, or UI.

See [`docs/agents.md`](docs/agents.md) for when and how to use them.
